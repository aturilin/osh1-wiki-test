# Отчёты в LMS: какие есть и как пользоваться

В LMS есть встроенная система отчётов, а для продвинутой аналитики мы используем Metabase. Эта статья поможет разобраться, какие данные доступны, где их найти и как ими пользоваться.

---

## Встроенные отчёты LMS

Встроенные отчёты доступны в веб-версии LMS: **Панель управления → Отчёты**.

> Доступ к отчётам есть у ролей: администратор, куратор (только по своей группе), методист. Учителя видят только свою статистику. Если вам нужен доступ к отчётам, но его нет — обратитесь к Игорю Ермолаеву.

### 1. Отчёт по посещаемости

**Путь:** Панель управления → Отчёты → Посещаемость

**Что показывает:**
- Процент посещения живых уроков по каждому ученику
- Сводка по классу / группе
- Динамика посещаемости по неделям

**Фильтры:**
- Период (неделя / месяц / квартал / произвольный)
- Класс
- Предмет
- Учитель
- Куратор (группа)

**Как читать отчёт:**

| Показатель | Зелёный (хорошо) | Жёлтый (внимание) | Красный (проблема) |
|------------|------------------|--------------------|--------------------|
| Посещаемость ученика | >80% | 60–80% | <60% |
| Средняя по классу | >75% | 60–75% | <60% |

[скриншот: отчёт по посещаемости — таблица учеников с цветовой индикацией]

**Что делать с данными:**
- Куратор: если ученик в «красной зоне» — связаться с родителями, выяснить причину
- Методист: если у конкретного учителя низкая посещаемость — проверить качество уроков
- Руководитель: отслеживать тренд — если посещаемость падает по всей школе, это системная проблема

### 2. Отчёт по успеваемости

**Путь:** Панель управления → Отчёты → Успеваемость

**Что показывает:**
- Средний балл ученика по каждому предмету
- Динамика оценок (рост / падение)
- Количество невыполненных ДЗ

**Фильтры:** период, класс, предмет, учитель, куратор.

**Как используется:**
- Кураторы: формирование отчёта для родителей (ежемесячный «табель»)
- Методисты: оценка сложности программы (если средний балл по предмету <3.5 — программа может быть слишком сложной)
- HR: данные для KPI учителей (см. hr-kpi.md)

### 3. Отчёт по активности учеников

**Путь:** Панель управления → Отчёты → Активность

**Что показывает:**
- Последний вход ученика в LMS
- Количество просмотренных записей
- Количество сданных ДЗ
- Активность в чате

**Зачем нужен:**
- Выявление «спящих» учеников — тех, кто оплатил, но не пользуется платформой. Это сигнал будущего отвала.
- Если ученик не заходил >7 дней — куратор получает автоматическое уведомление.

[скриншот: отчёт по активности — список учеников с датой последнего входа]

### 4. Отчёт по нагрузке учителей

**Путь:** Панель управления → Отчёты → Учителя → Нагрузка

**Что показывает:**
- Количество уроков в неделю у каждого учителя
- Количество учеников
- Средняя оценка (рейтинг)
- Количество непроверенных ДЗ

**Фильтры:** период, предмет, класс.

**Зачем нужен:**
- Координаторы: балансировка нагрузки (если у учителя >25 уроков в неделю — он перегружен)
- Методисты: контроль качества (если у учителя много непроверенных ДЗ — нужен разговор)
- HR: данные для пересмотра ставок

**Рекомендуемая нагрузка:**

| Тип | Мин. | Оптимально | Макс. |
|-----|------|------------|-------|
| Уроков/неделю | 10 | 15–20 | 25 |
| Учеников (уникальных) | 20 | 40–60 | 80 |

### 5. Финансовый отчёт

**Путь:** Панель управления → Отчёты → Финансы

**Доступ:** только администраторы + руководитель финансов (Анастасия Журавлёва).

**Что показывает:**
- Поступления за период (разбивка по тарифам)
- Количество активных подписок
- Количество отмен (отвал)
- MRR (Monthly Recurring Revenue)
- Средний чек
- LTV (Lifetime Value) — средняя выручка на ученика за всё время

**Этот отчёт — базовый.** Для детальной финансовой аналитики используйте Metabase (см. ниже).

## Экспорт данных

Все встроенные отчёты можно экспортировать в Excel (.xlsx):

1. Откройте нужный отчёт
2. Примените фильтры
3. Нажмите кнопку **«Экспорт»** (правый верхний угол)
4. Выберите формат: Excel или CSV
5. Файл скачается

[скриншот: кнопка «Экспорт» в интерфейсе отчёта]

> Экспорт ограничен 10 000 строк. Если нужно больше — обратитесь к Павлу Дроздову, он выгрузит напрямую из БД.

## Metabase: продвинутая аналитика

### Что такое Metabase

Metabase — это open-source BI-платформа, которую мы используем для построения дашбордов и сложных отчётов. Подключена к read-only реплике базы данных LMS.

**URL:** metabase.online-school1.ru (доступен только из корпоративной сети / VPN)

**Кто настраивает:** Павел Дроздов (подрядчик-аналитик, 45 000₽/мес). Павел создаёт дашборды по запросу руководителей.

### Доступ к Metabase

| Роль | Уровень доступа |
|------|----------------|
| Руководители отделов | Просмотр дашбордов своего отдела |
| Евгения Берёзина | Полный доступ (админ) |
| Павел Дроздов | Полный доступ (настройка) |
| Остальные сотрудники | Нет доступа (по запросу) |

**Как запросить доступ:** напишите Евгении Берёзиной в Тайм. Обоснуйте, зачем нужен доступ и к каким данным.

### Существующие дашборды

| Дашборд | Что показывает | Для кого |
|---------|---------------|----------|
| **Продажи — Воронка** | Конверсия по этапам, лиды по источникам, средний чек | Кирилл Лазарев, менеджеры |
| **Продажи — Динамика** | MRR, новые ученики, отвал, LTV — по месяцам | Руководство |

| **Учителя — Рейтинг** | Рейтинг учителей, посещаемость, ДЗ | Сергей Киреев |

[скриншот: главная страница Metabase — список дашбордов]

### Как заказать новый дашборд

1. Сформулируйте, что хотите видеть (какие метрики, фильтры, визуализация)
2. Напишите запрос Павлу Дроздову через Евгению Берёзину (или в канал #analytics в Тайм)
3. Павел оценит сложность и сроки (обычно 1–5 рабочих дней)
4. После готовности — ревью с вами и доработка

> **Совет:** перед тем как заказывать дашборд — посмотрите, может, нужные данные уже есть во встроенных отчётах LMS. Metabase — для случаев, когда встроенных отчётов недостаточно (кросс-срезы, сложные фильтры, визуализации).

### Как пользоваться Metabase

**Просмотр дашборда:**
1. Войдите на metabase.online-school1.ru (логин = корп. почта)
2. На главной странице — список доступных дашбордов
3. Нажмите на нужный дашборд
4. Используйте фильтры вверху (период, отдел, менеджер и т.д.)
5. Наведите на график — увидите детали

**Экспорт из Metabase:**
- Каждый график/таблицу можно скачать: наведите курсор → иконка «⬇» → CSV / Excel
- Весь дашборд — кнопка «Экспорт» в правом верхнем углу → PDF

**Создание своего запроса (для продвинутых):**
Если у вас есть навыки SQL — Metabase позволяет писать свои запросы. Но делайте это аккуратно: запросы идут к read-only реплике, навредить production вы не можете, но тяжёлые запросы могут нагрузить реплику.

## Частые вопросы

**Q: У меня нет раздела «Отчёты» в LMS. Почему?**
A: Этот раздел доступен только для ролей: администратор, куратор, методист. Если вам нужен доступ — обратитесь к Игорю Ермолаеву.

**Q: В отчёте неправильные данные. Что делать?**
A: Сначала проверьте фильтры — возможно, выбран не тот период или класс. Если фильтры верные и данные всё равно некорректны — напишите в #tech-support с описанием проблемы и скриншотом.

**Q: Мне нужен отчёт, которого нет в LMS. Как быть?**
A: Два варианта: (1) запросить дашборд в Metabase через Павла Дроздова, (2) попросить Евгению Берёзину добавить отчёт в LMS (если он нужен регулярно и многим).

**Q: Как часто обновляются данные в Metabase?**
A: Реплика БД обновляется каждые 15 минут. Данные в Metabase — практически в реальном времени (задержка до 15 мин).

**Q: Могу ли я создать свой дашборд в Metabase?**
A: Технически — да, если у вас есть доступ с правами создания. Но рекомендуем делать это через Павла — он знает структуру БД и сделает быстрее и правильнее.

**Q: Отчёт не экспортируется в Excel. Ошибка.**
A: Вероятно, слишком много данных (>10 000 строк). Сузьте фильтры или обратитесь к Павлу для прямой выгрузки.
